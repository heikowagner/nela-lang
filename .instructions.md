## 📋 Code Organization Rules

### Rule 1: All NELA Files Use LLM-Optimized Headers

Canonical format and regeneration behavior are defined in `.github/skills/nela-headers/SKILL.md`.
This file keeps the project-specific workflow and boundaries.

**Every .nela file in `examples/` has:**
- **Header Index** (lines 1-30): navigable section map with line numbers
- **Section Markers** (throughout): `@SECTION_*` with metadata (functions, purpose, dependencies)
- **Flat structure**: No imports; all functionality in one file per module

**Why?** NELA has no module/import system. Section indexes compensate for file size and enable LLM-friendly navigation.

**Pattern across all files:**
```nela
-- SECTION INDEX FOR LLM NAVIGATION (auto-maintained):
--   SECTION_MAIN      [start-end]    func1 | func2
--   SECTION_UTILITY   [start-end]    func3 | func4
```

### Rule 2: Section Declaration Order (All Files)

Every `@SECTION_*` marker has metadata comments:

```nela
-- ── @SECTION_NAME [start-end] ────────────────────────────────
-- Functions: func1, func2, func3
-- Purpose: Clear 1-line description
-- Called_by: other_section_or_function
-- Key_vars: [var1, var2]
-- Details: (optional deep explanation)
```

**LLM Must:** Maintain this metadata when editing any NELA file sections.

### Rule 3: No Forward References in Any File

Functions are declared in dependency order within each file. You cannot use a function before it is defined.

**Validator catches this** → `make check-header` fails if violated on any NELA file.

### Rule 4: Regenerate Headers After Changes

**When to regenerate header indexes:**

- Added/removed/renamed functions in a section → `make check-header && make fix-header`
- Changed section boundaries → `make fix-header` auto-detects changes
- Never manually edit the header index; use the validator tool (works on all files)

### Rule 5: Enforce Architecture Boundaries

```
✓ Allowed: 
  - All logic in NELA-S (game, algorithms, VM, utilities)
  - Python: input(), pygame framebuffer, color mapping

✗ Forbidden: 
  - Game logic in Python
  - Physics, AI, rendering outside NELA-S
```

## 🧪 Header Validation Workflow (MANDATORY)

### Before You Commit:

```bash
# 1. Check if headers are synchronized in ALL NELA files
make check-header

# 2. If `make check-header` FAILS on any file:
make fix-header          # Auto-regenerates headers for all files
git add examples/*.nela

# 3. If `make fix-header` succeeded, verify tests pass:
make test               # Run regression test suite (all files involved)
```

### When Header Is Out of Sync

**Scenario:** You edited any .nela file, added/removed functions, or changed section boundaries.

```
✗ ERROR: SECTION_ENEMIES header says [397-470], actual code is [397-480]
         (or any other file: stack_vm.nela, wolf_grid.nela, etc.)

→ Run: make fix-header
→ Verify: git diff examples/*.nela (only headers changed, no logic)
→ Commit: The regenerated headers
```

### Common Causes of Header Desync (Any File):

1. **Added function to a section** → section size changed
2. **Moved a function between sections** → two section boundaries changed
3. **Deleted/renamed functions** → function list changed
4. **Changed Comments/whitespace in section** → section size changed

## 📐 Enforced Invariants

When editing game logic, verify these invariants:

| Invariant | Check |
|-----------|-------|
| No forward references | `make check-header` verifies call graph |
| NELA-S owns all logic | Search `wolf_player.py` for no game logic |
| Flat map (1D index) | Verify index = `x + y*w`, never 2D array |
| State = `[px, py, angle]` | Check all state handlers use 3 fields |
| Enemy LOS = `is_wall_fat` | No new raycasting for LOS |
| Angles normalized [0,359] | Verify `norm_angle` is called on all angles |
| Floats for position | Check coordinate types consistently |

## 🆘 Troubleshooting

### ❌ `make check-header` fails

```
SECTION_ENEMIES header says [397-470], actual code is [397-489]
```

**Fix:**
```bash
make fix-header                    # Regenerate header
make test                          # Ensure no logic errors
git add examples/wolf_game.nela    # Add the updated header
```

### ❌ `make test` fails

```
Wolf Game:  PASS
V0.7:       FAIL    [error parsing wolf_game.nela]
```

**Likely causes:**
- Syntax error in NELA code (close parens? bad function call?)
- Broke one of the invariants (forward ref, wrong state format, etc)

**Fix:**
```bash
cd src
python3 -c "from nela_parser import parse_file; parse_file('../examples/wolf_game.nela')" 
# Will show parse error with line number
```

### ❌ Python runtime crashes

```
File "src/wolf_player.py", line X, in _split_payload()
  ValueError: not enough values to unpack
```

**Likely causes:**
- `render_packet` in NELA-S changed its output structure
- `wolf_player.py` is out of sync (too rigid; too many unpacks)

**Fix:**
- Update `_split_payload()` in `wolf_player.py` to match new packet format
- Re-run `python3 src/wolf_player.py` to test
